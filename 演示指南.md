## 一、项目一句话定位

> **智学助手：朴素贝叶斯用于文档分类 + 本地语义检索用于问答**

```
➢ 文档分类：Multinomial NB + 词袋模型 → 学科分类
➢ 智能问答：Embedding + FAISS HNSW → 本地语义检索
```

---

## 二、AI 方法核心说明（答辩必背）

本项目使用了**两类机器学习方法**，分别解决不同任务：

### 方法一：朴素贝叶斯分类器

| 问题 | 回答 |
|------|------|
| **解决什么问题？** | **分类** —— 将上传的文档自动归类到正确学科 |
| **为什么选该方法？** | 传统机器学习，无需 GPU，本地 CPU 即可运行；相比关键词规则，能综合考虑词汇分布，泛化能力更强 |
| **系统中的位置？** | `ai_module/document_classifier.py` → `DocumentClassifier` 类 |
| **输入是什么？** | 文档文本内容（字符串） |
| **输出是什么？** | 学科类别（如"计算机"）+ 置信度（0~1） |

**核心公式：**
```
P(类别|文档) ∝ P(类别) × ∏ P(词|类别)
```

---

### 方法二：本地语义检索（简化版 RAG）

| 问题 | 回答 |
|------|------|
| **解决什么问题？** | **检索** —— 根据用户问题，从知识库中找到相关文档片段 |
| **为什么选该方法？** | 相比关键词匹配（TF-IDF），Embedding 向量检索能捕捉语义相似性（如"深度学习"和"神经网络"即使词不重叠也能匹配）；完全本地运行，不依赖外部 API |
| **系统中的位置？** | `ai_module/rag_system.py` → `RAGSystem` 类 |
| **输入是什么？** | 用户问题文本（字符串） |
| **输出是什么？** | 相关文档片段 + 来源列表 |

**技术流程：**
```
用户问题 → BGE-small-zh 向量化 → FAISS HNSW 检索 → 返回相关片段
```

## 六、常见问题应答

**Q: 为什么不用大模型（如 KIMI）？**
> 答辩版本使用本地语义检索，完全离线运行，不依赖外部 API。大模型方案需要配置 KIMI Key，本地方案更适合演示环境。

**Q: 朴素贝叶斯的优点是什么？**
> 1. 算法简单，易于理解
> 2. 训练速度快，无需迭代优化
> 3. 可在 CPU 上运行，适合小型项目
> 4. 对小规模数据也能有不错的效果

**Q: 什么是词袋模型？**
> 将文档表示为词汇出现次数的向量，忽略语法和词序。如"机器学习是学习" → {机器:1, 学习:2, 是:1}。

**Q: Embedding 和关键词匹配的区别？**
> 关键词匹配只能找到包含相同词的文档；Embedding 能捕捉语义相似性，如"深度学习"和"神经网络"即使用词不同也能匹配成功。
