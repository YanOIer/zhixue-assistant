## 一、项目一句话定位

> **智学助手：朴素贝叶斯用于文档分类 + 本地语义检索用于问答**

```
➢ 文档分类：Multinomial NB + 词袋模型 → 学科分类
➢ 智能问答：BGE-small-zh Embedding + FAISS HNSW → 本地语义检索
```

---

## 二、AI 方法核心说明（答辩必背）

本项目使用了**两类机器学习方法**，分别解决不同任务：

### 方法一：朴素贝叶斯分类器

| 问题 | 回答 |
|------|------|
| **解决什么问题？** | **分类** —— 将上传的文档自动归类到正确学科（数学/英语/政治/计算机） |
| **为什么选该方法？** | 传统机器学习，无需 GPU，本地 CPU 即可运行；相比关键词规则，能综合考虑词汇分布，泛化能力更强 |
| **系统中的位置？** | `ai_module/document_classifier.py` → `DocumentClassifier` 类 |
| **输入是什么？** | 文档文本内容（字符串） |
| **输出是什么？** | 学科类别（如"计算机"）+ 置信度（0~1） |

**核心公式：**
```
P(类别|文档) ∝ P(类别) × ∏ P(词|类别)
```

**技术细节：**
- 特征表示：词袋模型（BOW）+ TF-IDF
- 训练数据：每个类别 20 篇（关键词自动生成）
- 拉普拉斯平滑：α=1.0，避免零概率

---

### 方法二：本地语义检索（简化版 RAG）

| 问题 | 回答 |
|------|------|
| **解决什么问题？** | **检索** —— 根据用户问题，从知识库中找到相关文档片段 |
| **为什么选该方法？** | 相比关键词匹配（TF-IDF），Embedding 向量检索能捕捉语义相似性；完全本地运行，不依赖外部 API |
| **系统中的位置？** | `ai_module/rag_system.py` → `RAGSystem` 类 |
| **输入是什么？** | 用户问题文本（字符串） |
| **输出是什么？** | 相关文档片段 + 来源列表 |

**技术流程：**
```
用户问题 → BGE-small-zh 向量化 → FAISS HNSW 检索 → 返回相关片段
```

**技术细节：**
- 向量化模型：BGE-small-zh（24M 参数，130MB，专为中文设计）
- 向量索引：FAISS HNSW（Hierarchical Navigable Small World）
- 分块策略：滑动窗口（chunk_size=500，overlap=100）

## 三、关键概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **Embedding** | 把文字变成数字向量，让计算机能用数学方法比较"语义相似度" |
| **FAISS HNSW** | Facebook 开发的向量检索库，用"高速公路网络"快速找相似向量 |
| **词袋模型** | 将文档表示为词汇出现次数，忽略语法和词序 |
| **TF-IDF** | 词频-逆文档频率，衡量词语在文档中的重要性 |
| **SQLite** | 轻量级数据库，零配置、单文件，适合课程演示 |

## 七、常见问题应答

**Q: 为什么选择朴素贝叶斯？**
> 1. 算法简单，适合课程项目
> 2. 无需 GPU，CPU 即可运行
> 3. 对小规模数据效果好
> 4. 相比关键词规则，能综合考虑词汇分布

**Q: 为什么不用大模型（如 KIMI）？**
> 答辩版本使用本地语义检索，完全离线运行，不依赖外部 API。大模型方案需要配置 Key 且响应较慢，本地方案更适合答辩演示。

**Q: 什么是词袋模型？**
> 将文档表示为词汇出现次数的向量，忽略语法和词序。
> 例："机器学习是学习" → {机器:1, 学习:2, 是:1}

**Q: Embedding 和关键词匹配的区别？**
> 关键词匹配只能找到包含相同词的文档；Embedding 能捕捉语义相似性。
> 例："深度学习"和"神经网络"词不重叠，但语义相近，Embedding 也能匹配。

**Q: 为什么选 BGE-small-zh？**
> 专为中文设计，参数量仅 24M，能在 CPU 上流畅运行，模型大小仅 130MB。

**Q: 什么是 FAISS HNSW？**
> Facebook 开发的向量检索库。HNSW 是"分层可导航小世界图"，用"高速公路网络"思想快速定位最近邻。
