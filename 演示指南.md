## ⚡ 一分钟启动

### 选择启动模式

本项目有两种后端模式：

| 模式 | 脚本 | 特点 | 适合场景 |
|------|------|------|---------|
| **RAG 完整模式** | `start_rag.bat` | 加载 RAG 模型，语义检索回答 | **推荐！答辩演示用** |
| **简单模式** | `start_simple.bat` | 不加载 RAG，本地关键词检索 | 仅前端调试 |

**请使用 `start_rag.bat`**（RAG 完整模式）进行答辩演示。

### RAG 完整模式（RAG 模式，推荐）

双击 `start_rag.bat`，等待出现：
```
[4/4] 启动 FastAPI 服务...
============================================================
  🎉 智学助手 RAG 完整模式已启动！
  API地址: http://localhost:8000
  API文档: http://localhost:8000/docs
============================================================
```
即可。不要关闭这个窗口。

**RAG 模式会自动：**
1. 加载本地 Embedding 模型（BAAI/bge-small-zh）
2. 从数据库读取已上传文档并重建向量索引
3. 启用本地语义检索（RAG 向量检索，无需配置 API Key）

### 第二步：打开微信开发者工具

1. 打开微信开发者工具
2. 点击「导入项目」
3. 项目目录选择：`d:\Code\Projects\zhixue-assistant\frontend`
4. AppID 填写已有的测试号或点击「测试号」
5. 点击确定，等待编译完成

> ⚠️ **重要设置**：右上角「详情」→「本地设置」→ 勾选「**不校验合法域名**」

## 🤖 机器学习/人工智能方法说明（答辩核心）

本系统使用了两类机器学习方法，分别解决不同任务：

---

### 方法一：朴素贝叶斯分类器（文档自动分类）

| 项目 | 内容 |
|------|------|
| **解决什么问题** | **分类** —— 将上传的文档自动归类到正确学科（数学/英语/政治/计算机/其他） |
| **为什么选择该方法** | 传统机器学习算法，小型数据集也能训练，无需 GPU，可本地 CPU 运行；相比规则方法（关键词匹配），朴素贝叶斯能综合考虑词汇分布，泛化能力更强 |
| **在系统中的位置** | `ai_module/document_classifier.py`，`DocumentClassifier` 类 |
| **输入** | 文档文本内容（字符串，最长不限） |
| **输出** | 分类结果：类别名称（如"计算机"）+ 置信度（0~1之间） |

**技术细节：**
- 算法：Multinomial Naive Bayes（多项式朴素贝叶斯）
- 特征表示：词袋模型（BOW），中英文分词 + 字级别特征
- 训练数据：基于领域关键词自动生成（每个类别20条样本）
- 拉普拉斯平滑参数：α=1.0，避免零概率问题
- 推理公式：`P(类别|文档) ∝ P(类别) × ∏P(词|类别)`

**答辩演示重点：**
上传任意文档后，弹窗中会显示 **"分类：计算机 / 置信度：XX%"**，说明文档分类器自动识别了文档所属学科。

---

### 方法二：RAG 检索系统（本地语义检索问答）

| 项目 | 内容 |
|------|------|
| **解决什么问题** | **检索 + 生成** —— 根据用户问题，从知识库中检索相关文档片段并返回答案 |
| **为什么选择该方法** | 相比纯关键词检索（TF-IDF/BM25），Embedding 向量检索能捕捉语义相似性（"深度学习"和"神经网络"即使词不重叠也能匹配）；相比直接调用大模型，RAG 保证了答案可溯源，不依赖外部 API |
| **在系统中的位置** | `ai_module/rag_system.py`，`RAGSystem` 类 |
| **输入** | 用户问题文本（字符串） |
| **输出** | 回答内容 + 参考来源文档列表 + 匹配的文档片段 |

**技术细节：**

```
用户提问 → BGE-small-zh 向量化 → FAISS HNSW 检索 → 匹配片段 → 返回答案
```

- **文本向量化**：BAAI/bge-small-zh（中英文语义 Embedding 模型，约 130MB，本地缓存）
- **向量检索**：FAISS HNSW 近似最近邻索引（`IndexHNSWFlat`，搜索速度快）
- **分块策略**：滑动窗口分块（chunk_size=500，overlap=100），保证语义连贯

**系统架构图：**
```
上传文档
    ↓
[文档处理] process_document() — 提取文本
    ↓
[文档分类] DocumentClassifier — Naive Bayes —→ 分类标签
    ↓
[文本分块] _split_text() — 滑动窗口切分
    ↓
[向量化] BGE-small-zh Embedding —→ 向量
    ↓
[索引存储] FAISS HNSW —→ vector_index.faiss
    ↓
用户提问
    ↓
[向量化问题] BGE-small-zh —→ 查询向量
    ↓
[相似度检索] FAISS.search() —→ Top-K 相关片段
    ↓
[返回结果] 答案 + 来源列表
```

**答辩演示重点：**
进入「对话」Tab 输入问题如"什么是监督学习？"，展示：
- 回答内容
- 📌 **本地知识库检索** 标签
- 参考来源列表（文档名）

## 🔧 常见问题

**Q: 小程序报"网络请求失败"**
→ 确认后端在运行（8000端口），检查「不校验合法域名」已勾选

**Q: 上传文件后分类显示"其他"**
→ 正常，上传纯数字/空白文件会分类到"其他"；有实际内容的文档会正确分类

**Q: 问答回答说"没有匹配内容"**
→ 先上传相关文档，再提问；关键词要出现在文档中

**Q: 后端启动报错**
```bash
pip install fastapi uvicorn python-multipart scikit-learn PyPDF2
```
